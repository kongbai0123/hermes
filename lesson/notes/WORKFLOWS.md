# 實戰工作流

## 查錯流程

1. 先跑測試取得失敗輸出。
2. 搜尋錯誤函式、類別或關鍵字。
3. 讀取最小必要檔案。
4. 讓 agent 解釋錯誤原因。
5. 產生 patch。
6. 人工審查 patch。
7. 套用後再跑測試。

目前第一版只做到第 5 步：產生 patch，不直接套用。

## Prompt 工作流

好的 prompt 要說清楚：

- agent 的角色
- 可用工具
- 禁止事項
- 輸出格式
- 完成條件

範例：

```text
你是本機程式碼助理。只能讀取 workspace 內檔案。
如果需要工具，請輸出 JSON action。
如果要修改，請只產生 unified diff，不要宣稱已修改。
```

## 安全工作流

第一版 agent 的安全重點：

- 路徑要經過 `safe_path`
- 命令要經過白名單
- 不開放刪除工具
- patch 預設不套用
- 工具呼叫要印出來，方便觀察
